    Mermaid Diagram Style Guide¶

    This guide provides standards for creating consistent Mermaid diagrams in the Music Besties documentation.

    Diagram Types and Usage¶

    Diagram Type Best Used For Example Usage
    Flowchart Component relationships, processes Application architecture, component hierarchy
    Sequence Interaction flows, API calls User interactions, data flow between components
    Class Data structures, object models Data management, state structure
    Gantt Project timelines Implementation plans, sprint schedules
    State State transitions User flows, application states

    Color Standards¶

    Use these standardized colors for consistency across all diagrams:

    Element Type Color Code Usage
    Primary/Main #5D87E1 (Indigo blue) Main components, primary actions
    Secondary #FF8A65 (Coral orange) Secondary components, user actions
    Tertiary #82B1FF (Light blue) Supporting elements
    Accent #7CB342 (Green) Success states, completed actions
    Warning #FFB74D (Orange) Warning states, caution actions
    Error #E57373 (Red) Error states, destructive actions

    Styling Examples¶

    Flowchart Example¶

    flowchart TD
    A[Primary Component] --> B[Secondary Component]
    A --> C[Tertiary Component]

    classDef primary fill:#5D87E1,stroke:#333,stroke-width:1px,color:white
    classDef secondary fill:#FF8A65,stroke:#333,stroke-width:1px,color:white
    classDef tertiary fill:#82B1FF,stroke:#333,stroke-width:1px,color:white

    class A primary
    class B secondary
    class C tertiary

    Sequence Diagram Example¶

    sequenceDiagram
    participant User
    participant App as Application
    participant DB as Database

    User->>App: Action
    App->>DB: Query
    DB-->>App: Response
    App-->>User: Update UI

    Class Diagram Example¶

    classDiagram
    class MainClass {
        +String property
        +method() void
    }
    class RelatedClass {
        +String property
        +method() void
    }
    MainClass --> RelatedClass

    Best Practices¶

    1. Keep diagrams simple - Focus on the key information you want to convey
    2. Use consistent naming - Use the same terminology across all diagrams
    3. Add descriptive labels - Make edge labels descriptive but concise
    4. Include a caption - Always add a markdown caption below the diagram
    5. Test responsiveness - Ensure diagrams render well on different screen sizes

    Accessibility Considerations¶

    • Ensure sufficient color contrast for all elements
    • Don't rely solely on color to convey information
    • Add descriptive text captions for all diagrams
    • Use patterns or shapes in addition to colors when possible

    Technical Implementation¶

    The global Mermaid configuration provides: - Consistent styling across all diagrams - Dark mode support - Responsive rendering - Interactive features (node highlighting on click)

    When creating new diagrams, you can rely on the global styling and only add custom classes when necessary for specific diagram elements.
